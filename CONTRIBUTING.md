# راهنمای مشارکت (Contributing Guide)

این فایل گام‌به‌گام توضیح می‌دهد که چطور پروژه خود را به این مخزن اضافه کنید — همه مراحل **فقط در مرورگر و محیط وب گیت‌هاب** انجام می‌شود.

---

## گام ۱ — Fork کردن مخزن اصلی

**Fork** یعنی یک نسخه کامل از این مخزن زیر اکانت گیت‌هاب خودتان ساخته می‌شود.

1. در صفحه اصلی این مخزن، دکمه **Fork** را در گوشه بالا-راست بزنید
2. گیت‌هاب چند ثانیه صبر می‌کند و بعد شما را به مخزن جدیدتان هدایت می‌کند:

```
https://github.com/YOUR-USERNAME/iran-market-analysis
```

> ✅ از این لحظه، مخزن روی اکانت شما است و می‌توانید آزادانه تغییر بدهید.

---

## گام ۲ — ساخت Branch اختصاصی

هرگز مستقیم روی شاخه `main` کار نکنید. یک Branch جداگانه برای پروژه خودتان بسازید.

1. در مخزن Fork شده خودتان، روی منوی **main ▾** (بالا-چپ فهرست فایل‌ها) کلیک کنید
2. در کادر جستجو، نام Branch جدید خود را بنویسید:

```
analysis/نام-شما-موضوع
```

**نمونه‌های درست:**
```
analysis/ali-peugeot-207
analysis/sara-iphone13-price
analysis/reza-tehran-west-rent
analysis/maryam-asus-gaming-laptop
```

3. گزینه **Create branch: analysis/... from main** را کلیک کنید

> ✅ حالا روی Branch جدید خود هستید. نام آن را در منوی بالا-چپ می‌بینید.

---

## گام ۳ — ساخت پوشه پروژه

1. وارد پوشه `projects/` شوید (روی آن کلیک کنید)
2. دکمه **Add file ▸ Create new file** را بزنید
3. در کادر نام فایل بنویسید:

```
YOUR-NAME/README.md
```

> نوشتن `/` در نام فایل به‌صورت خودکار پوشه می‌سازد.

4. محتوای فایل README را بنویسید (قالب پایین را کپی کنید)
5. دکمه **Commit changes** را بزنید — پیام Commit گویا بنویسید:

```
Add project folder for Ali - Peugeot 207 analysis
```

---

## گام ۴ — آپلود فایل‌ها

### ۴.۱ — آپلود Notebook و Dataset

1. وارد پوشه‌ای که ساختید بشوید (`projects/نام-شما/`)
2. دکمه **Add file ▸ Upload files** را بزنید
3. فایل‌های زیر را آپلود کنید:

| فایل | توضیح | اجباری؟ |
|------|-------|---------|
| `notebook.ipynb` | نوت‌بوک اصلی با کد Cleaning و EDA | ✅ بله |
| `dataset_sample.csv` | نمونه داده (حداقل ۵۰۰ سطر) | ✅ بله |
| `charts/chart1.png` | تصاویر نمودارها | ✅ حداقل ۳ نمودار |

4. در کادر **Commit changes** پیام مناسب بنویسید:

```
Add initial notebook and dataset for Peugeot 207 analysis
```

### ۴.۲ — چند نکته درباره Commit

- هر بار که فایلی آپلود یا ویرایش می‌کنید، یک Commit ثبت می‌شود
- لازم نیست همه کار را یکجا انجام دهید — می‌توانید چند Commit داشته باشید
- **پیام Commit خوب:**
  ```
  Add EDA charts: price distribution and mileage correlation
  Fix: convert Persian numerals to English in price column
  Update README with key findings and chart images
  ```
- **پیام Commit بد:**
  ```
  update
  fix
  changes
  ```

## گام ۵ — باز کردن Pull Request

وقتی همه فایل‌ها آماده بود:

1. گیت‌هاب یک بنر زرد نشان می‌دهد:  
   **«Your branch is ahead of main — Compare & pull request»**  
   روی آن کلیک کنید.

2. اگر بنر نبود، تب **Pull requests** را باز کنید و **New pull request** را بزنید.

3. تنظیمات PR را چک کنید:

```
base repository: [نام-استاد]/iran-market-analysis   base: main
head repository: [نام-شما]/iran-market-analysis      compare: analysis/نام-شما-موضوع
```

4. **عنوان PR** را طبق این قالب بنویسید:

```
[تحلیل] نام-شما | موضوع پروژه
```

مثال:
```
[تحلیل] علی احمدی | قیمت پژو ۲۰۷ دست دوم در دیوار
```

5. قالب توضیحات PR به‌صورت خودکار پر می‌شود. تمام بخش‌ها را کامل کنید.

6. دکمه **Create Pull Request** را بزنید.

> ✅ تمام! PR شما در صف بررسی استاد است.

---

## گام ۶ — پاسخ به بازخورد استاد

- استاد در تب **Files changed** روی خطوط کد کامنت می‌گذارد
- شما نوتیفیکیشن ایمیل دریافت می‌کنید
- تغییرات لازم را اعمال کنید و Commit جدید بزنید — PR به‌صورت خودکار آپدیت می‌شود
- وقتی استاد **Approve** کرد، PR شما Merge می‌شود

---

## گام ۷ — بعد از Merge

پس از Merge شدن PR:

1. به فایل `README.md` اصلی مخزن استاد بروید
2. روی آیکون مداد (Edit) کلیک کنید
3. ردیف خودتان را به جدول **پروژه‌های دانشجویان** اضافه کنید:

```markdown
| علی احمدی | قیمت پژو ۲۰۷ | دیوار | ✅ Merged | [#5](link) |
```

4. یک Commit جدید بزنید و PR دوم کوچک باز کنید.

---

## ❓ سوالات متداول

**می‌توانم بعد از باز کردن PR فایل اضافه کنم؟**  
بله. هر Commit جدید روی Branch شما به‌صورت خودکار به PR اضافه می‌شود.

**اگر اشتباهی فایل را در پوشه اشتباه آپلود کردم چه کار کنم؟**  
روی فایل کلیک کنید، آیکون سطل آشغال (Delete this file) را بزنید و Commit کنید.

**آیا می‌توانم Notebook را مستقیم در گیت‌هاب ویرایش کنم؟**  
گیت‌هاب فایل `.ipynb` را نمایش می‌دهد اما ویرایش مستقیم ندارد. کد را در Colab یا VS Code ویرایش کنید، سپس دوباره آپلود کنید.
